Models

new Backbone.Model()

What?

Models are the heart of any JavaScript application, containing the interactive data as well as a large part of the logic surrounding it: conversions, validation, computed properties, and access control.

backbonejs.org/#Model

An object to group data and functions related to the data together.

Examples

  • node
  • user
  • post
  • comment

How to create a model?


        var toon = new Backbone.Model({
          name: "Toon Ketels",
          age: 30,
          job: "Developer",
          bio: "
Scrum master, backend Drupal developer.
",
          uid: 7544
        });
        

        {
          _changing: false,
          _pending: false,
          _previousAttributes: {/*...*/},
          _attributes: {/*...*/},
          changed: {/*...*/},
          cid: "cid"
        };

What can we do with it?

Get attributes

        toon.get('age');
        toon.escape('bio');

Set attributes

        toon.changed;
        toon.previousAttributes();

Events

Hey, I've changed!

        toon.on('change', function(model) {
          console.log('changed');
          console.log(model);
        });
        

        toon.on('change:age', function(model, value) {
          console.log('Age has hanged');
          console.log(model);
          console.log(value);
        });

Views

new Backbone.View()

What?

Views are objects which do two things: 1) display data (most of the time models) and 2) capture user interaction to trigger code.

Examples

  • List of blogposts
  • A blogpost
  • A button to paginate

How to create a view?


        var view = new Backbone.View( {el: $('#main') } );

The model has changed, the view not.

'+this.model.get('name')+'
';

Keeping state

Person is selected. Selected is a "state", no?

        var PersonView = Backbone.View.extend({
        
          // ...
          
          events: {
            'click h2':           'selectPerson',
            // ...
          },
          
          selectPerson: function(event) {
            if (this.model.get('selected')) {
              this.model.set('selected', false);
            } else {
              this.model.set('selected', true);
            }
          },
          
          // ...
          
        });

Line of events

Going back and forth
  • User clicks `h2` title
  • View calls `selectPerson`
  • Function will toggle the `selected` attribute of the model
  • Model emits `change` event
  • Since the view's render function is tied to this event, `render` get's called
  • Render takes the model's attributes (some of them were just updated) and displays itself with the updated values.

        var TableRowView = Backbone.View.extend({

3 interesting things

  1. one model, two views
  2. view updates itself
  3. a different view updates the model

Got it?

  • Create a view
  • Set its contents
  • Insert it into the DOM
  • Pass models so the content is dynamic
  • Auto update its display when model changes
  • Use templates
  • Catch user interaction with "events" property

How does this help me organize my code?

  • All user interaction goes via Views, in its events hash
  • Separate data (model) from display (view) allows us to have multiple displays
  • Templates keep the views code tidy
  • By binding to model change event, the view automatically gets updated.
By binding to model change event, a view automatically gets updated.
A view does not care who updates the model or how.
One view can update the display of another view via a model, without even knowing it.
